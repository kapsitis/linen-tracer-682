Appendix: The C++ Environments

The preparation labs are sets of instructions defining the suggested configurations to run coding exercises and programming projects. By themselves they are not graded, but going through these steps is highly recommended (and you can also suggest improvements, if the same thing can be done better).

PLAB1: Build a C++ Project in Visual Studio
Create a directory structure for a small C++ project. Use one or more external C++ classes and their header files. Use the IFNDEF guards to include them properly. Learn about some Microsoft compiler options.
Why do we care? This setup can be used to do C++ coding on a Windows10 OS machine.
PLAB2: Build a C++ Project using Makefile
Compilation and linking options. Build variables and variable paths.
Why do we care? Grading server runs Makefile from Jenkins. To be sure that our project can run from the command-line, we can use the same Makefile.
PLAB3: Test a C++ Project in a VirtualBox Linux
Install VirtualBox software. Download and run an Xubuntu instance (optionally - set it up on your own). Run a minimalistic C++ development environment. Check out, run and test some C++ projects from the command-line, measure their running time.
Why do we care? Grading server uses "g++" and similar compilers in Linux; you can get advance knowledge, if something is not configured correctly. Virtual machines are helpful to compare runtime for different algorithms.
PLAB4: Use logging and IDE debugger
Apply "spdlog" library to create searchable logfiles. Use the debugger in Visual Studio or other IDE.
Why do we care? Debugger is just one way to fix errors, sometimes it is the best choice to understand what is going on.
PLAB5: Build a tiny package for R
Compile and use a C++ extension to R.
Why do we care? Sometimes algorithms in C++ are extending other development platforms to make them more efficient. R/RStudio is normally used for interdisciplinary and business applications, reporting and data visualizations.
